# Focus and Scope Instructions

## Core Principle
**Solve ONLY what is explicitly requested. Nothing more.**

## Ask if you get stuck
If you encounter a situation that seems unreasonably hard or even close to impossile, ask the user using the vscode/askQuestions tool.

## Scope Discipline
- Read the user's request twice before starting.
- Identify the EXACT problem to solve or task to work on.
- If the user says "do X", do X and stop.
- If the user says "keep it simple", that means minimal changes only.
- Never add unrequested features, improvements, or "helpful" additions unless explicitly asked.

## DO NOT do any of the following unless specifically requested:
- Adding functionality that was not requested
- "Improving" or "enhancing" existing systems that are out of scope
- Adding logging, warnings, or helpful messages beyond what's needed
- Creating unrequested new UI elements or controls without asking the user first
- Adding error handling unless specifically requested
- Implementing "best practices" that weren't asked for
- "Testing" code by starting or running it
- Committing code to version control

Note: building is of course allowed, since you have to check if the code compiles.

## Implementation Rules
- **No Scope Creep**: If you think "it would be nice to also...", don't
- **Respect Existing Patterns**: Don't change how things work unless its part of the task at hand.
- **Focus Check**: Before each addition, ask "Is this really part of the task that the user gave me?"

## When in Doubt
- Do less, not more
- Ask for clarification rather than assuming
- Stick to the exact wording of the request
- Remember: The user knows their codebase better than you do

**Bottom Line: Solve the stated problem efficiently and do not go overboard.**
